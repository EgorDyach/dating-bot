# Практика: Сравнение типов кеширования

## Описание

Этот проект реализует три различные стратегии кеширования на одной системе и сравнивает их производительность под разными нагрузками.

- **Lazy Loading (Cache-Aside)**: Чтение из кеша, промахи идут в БД, запись только в БД
- **Write-Through**: Одновременное писание в кеш и БД
- **Write-Back**: Писание в кеш, асинхронный флеш в БД каждые 5 секунд

## Структура проекта

```
cache-comparison/
├── src/                 # NestJS приложение
│   ├── lazy/           # Cache-Aside стратегия
│   ├── write-through/  # Write-Through стратегия
│   ├── write-back/     # Write-Back стратегия
│   └── metrics/        # Сбор метрик
├── load-generator/     # CLI для нагрузочного тестирования
├── scripts/            # SQL инициализация БД
└── results/            # Результаты тестов (JSON)
```

## Запуск

### Предварительно

```bash
# Перейти в папку проекта
cd cache-comparison

# Установить зависимости
npm install
cd load-generator && npm install && cd ..

# Поднять Docker контейнеры (PostgreSQL + Redis)
docker-compose up -d

# Дождитесь, когда БД будет готова (~10 сек)
docker-compose logs -f postgres
```

### Запуск приложения

```bash
# В режиме разработки с автоперезагрузкой
npm run start:dev

# Проверка здоровья
curl http://localhost:3000/health
```

### Запуск тестов

Один тест:
```bash
cd load-generator
npx tsx index.ts --strategy lazy --mode read-heavy --duration 30 --concurrency 10
```

Все 9 тестов (3 стратегии × 3 режима):
```bash
bash run-all-tests.sh
# Или с параметрами:
bash run-all-tests.sh 30 10  # duration=30s, concurrency=10
```

## Метрики

Для каждой стратегии собираются:

1. **Cache Hits/Misses** — попадания и промахи в кеш
2. **DB Reads/Writes** — количество обращений в БД
3. **Throughput** — количество успешных запросов в секунду
4. **Latency** — среднее время отклика, P95, P99

Метрики доступны по API:
```bash
# Все метрики
curl http://localhost:3000/metrics

# Для конкретной стратегии
curl http://localhost:3000/metrics?strategy=lazy
```

## Общий анализ

### Lazy Loading (Cache-Aside)

**Преимущества:**
- Гибкая стратегия инвалидации кеша
- Простая реализация
- Минимальное взаимодействие с кешем для записи

**Недостатки:**
- Cache Stampede при массовых промахах
- Дополнительная задержка при промахе

**Хорошо подходит для:**
- Частого чтения с редкой записью
- Когда данные могут быть невалидны в кеше

### Write-Through

**Преимущества:**
- Гарантированная согласованность кеша и БД
- Простая логика
- Хорошо для критичных данных

**Недостатки:**
- Запись медленнее (ждёт оба ответа)
- Может перегружать систему при high write load

**Хорошо подходит для:**
- Критичных данных, где важна согласованность
- Когда БД достаточно быстрая

### Write-Back

**Преимущества:**
- Самая быстрая запись
- Оптимизирует нагрузку на БД через batch
- Подходит для high write load

**Недостатки:**
- Риск потери данных при crash
- Сложнее в реализации
- Возможны несогласованности на время

**Хорошо подходит для:**
- High write load
- Non-critical данных (логирование, аналитика)
- Когда можно позволить себе потерять последние обновления

## Файлы результатов

Результаты каждого теста сохраняются в JSON в папке `results/`:

```bash
ls -la results/
# Примеры имён файлов:
# lazy-read-heavy-1715876543210.json
# write-through-balanced-1715876598456.json
# write-back-write-heavy-1715876653789.json
```

Каждый файл содержит:
- Конфигурацию теста
- Результаты load-generator (throughput, latencies)
- Метрики из приложения (cache hits/misses, DB calls)

## API Endpoints

### Lazy Loading
- `GET /lazy/:id` — Получить продукт
- `PUT /lazy/:id` — Обновить продукт

### Write-Through
- `GET /write-through/:id` — Получить продукт
- `PUT /write-through/:id` — Обновить продукт

### Write-Back
- `GET /write-back/:id` — Получить продукт
- `PUT /write-back/:id` — Обновить продукт (очередь)
- `GET /write-back/flush` — Ручной флеш в БД

### Утилиты
- `GET /metrics` — Все метрики
- `GET /metrics?strategy=lazy` — Метрики для конкретной стратегии
- `PUT /metrics/reset` — Сбросить метрики
- `GET /health` — Здоровье приложения
