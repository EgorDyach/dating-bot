# Dating Bot — учебный проект

Практика по разработке dating-приложения в связке с Telegram-ботом: анкеты, ранжирование, кэш, очереди, фоновые задачи, хранилище медиа.

## Статус этапов

| Этап | Содержание | Статус |
|------|------------|--------|
| 1 | Планирование: сервисы, архитектура, схема БД, репозиторий | Выполнено |
| 2 | Базовая функциональность: бот, `/start`, регистрация по Telegram ID | Выполнено |
| 3 | Анкеты, ранжирование, Redis, интеграция с ботом/MQ | Планируется |
| 4 | Celery, оптимизация, тесты, деплой | Планируется |

## Этап 2 — запуск бота

1. Создай бота у [@BotFather](https://t.me/BotFather), скопируй токен.
2. Подними PostgreSQL: при первом запуске контейнера схема подхватится из `docs/schema.sql` (`docker-entrypoint-initdb.d`):

   ```bash
   docker compose up -d
   ```

   Если том уже был создан без схемы, удали том или выполни вручную: `psql "$DATABASE_URL" -f docs/schema.sql`.

3. Установи зависимости и переменные окружения:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -r requirements.txt
   cp .env.example .env
   ```

   В `.env` укажи `BOT_TOKEN` и при необходимости `DATABASE_URL` (по умолчанию совпадает с `docker-compose.yml`).

4. Запуск из корня репозитория:

   ```bash
   python -m bot.main
   ```

Команды в Telegram: `/start` — upsert в таблице `users` по `telegram_id`, приветствие для нового и существующего пользователя; `/help` — краткая справка.

## Требования из ТЗ (напоминание)

- Рейтинг: минимум по одному пункту с каждого из трёх уровней (первичный, поведенческий, комбинированный).
- Рейтинги в отдельных таблицах, пересчёт через Celery; Redis для предварительно отранжированных списков (например, пачки по 10 анкет).
- Брокер сообщений для событий взаимодействий; метрики и логирование; S3-совместимое хранилище для изображений (например, MinIO).

Лицензия и состав команды — по договорённости с преподавателем.
